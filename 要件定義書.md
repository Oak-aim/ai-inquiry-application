# 要件定義書 
### プロジェクト名：AI Inquiry Application 
### 作成日：2026年5月14日 

## 1 システム概要 
## 1.1　背景・目的
```
現在、総務への問い合わせはメールや口頭で行われており、対応履歴の管理や過去回答の参照が困難となっている。

社員（一般ユーザー）からの問い合わせ内容をAIで自動分類し、緊急度判定および回答案生成を行うことで、問い合わせ対応の効率化を実現する。
```

## 1.2　システム構成
```
ユーザー（ブラウザ）
  ↓ (localhost:8501)
Streamlit（GUI）
  ↓ (localhost:8000)
FastAPI（APIサーバー） ➞  Gemini API（APIサーバー）
  ↓
ローカルJSONファイル（データ保存・書き込み）
```

```
| レイヤー | 技術 | 用途 |
|---|---|---|
| フロントエンド | Streamlit | 画面表示・ユーザー操作 |
| バックエンド | FastAPI | APIエンドポイント・業務ロジック |
| AI | Gemini API (google-genai) | カテゴリ分類・緊急度判定・回答案生成 |
| データ保存 | JSONファイル | タスクデータの永続化 |
| 仮想環境 | uv | Pythonパッケージ管理 |
| バージョン管理 | Git / Sourcetree | ソースコード管理 |
```

## 2 機能要件
- 問い合せ入力機能
- AIカテゴリー判定機能
- AI緊急度判定機能
- AI回答案生成機能
- 履歴保存機能
- 一覧表示機能
- 詳細表示機能

### 2.1 問い合せ入力機能
- ユーザーがテキストエリアに問い合わせ内容を入力できる
- 問い合わせ内容が未入力の場合はエラーメッセージを表示する
- 送信後に送信完了メッセージを表示すること

### 2.2 AIカテゴリー判定機能
- 問い合わせ内容を Gemini AI でカテゴリー判定して値を返す
- AI がカテゴリーを以下から1つ選択する
  - 勤怠
  - 休暇
  - 給与
  - 経費精算
  - 社員情報変更
  - その他

### 2.3 AI緊急度判定機能
- AIが問い合わせ内容から緊急度を判定する
- 緊急度は以下の3段階とする
  - 高
  - 中
  - 低

### 2.4 AI回答案生成機能
- AIが問い合わせ内容に対する回答案を生成する
- 生成した回答案を画面へ表示する

### 2.5 履歴保存機能
- 問い合わせ内容をJSONファイルへ保存する
- 保存項目は以下とすること
  - 問い合わせ内容
  - カテゴリ
  - 緊急度
  - 回答案
  - 登録日時

### 2.6 一覧表示機能
- 保存済み問い合わせを一覧表示できる
- 一覧には以下を表示する
  - 問い合わせ内容
  - カテゴリ
  - 緊急度
  - 回答案
  - 登録日時

### 2.7 詳細表示機能
- 問い合わせ詳細を表示できる
- 詳細画面には以下を表示する
  - 問い合わせ内容
  - カテゴリ
  - 緊急度
  - 回答案
  - 登録日時

## 3 非機能要件
### 3.1 実行環境
- 本システムは、ローカルPC環境上で正常に動作すること。

### 3.2 操作性
- 利用者がWebブラウザから直感的に操作できるユーザインタフェースを提供すること。

### 3.3 セキュリティ
- APIキーなどの機密情報はソースコードへ直接記述せず、環境変数等を用いて管理すること。

### 3.4 データ管理
- 問い合わせ内容および対応履歴をデータベースへ保存し、継続的に参照可能とすること。
