# 要件定義書 
### プロジェクト名：AI Inquiry Application 
### 作成日：2026年5月14日 

---

## 1. システム概要 
## 1.1　背景・目的
```
現在、総務への問い合わせはメールや口頭で行われており、対応履歴の管理や過去回答の参照が困難となっている。

社員（一般ユーザー）からの問い合わせ内容をAIで自動分類し、緊急度判定および回答案生成を行うことで、問い合わせ対応の効率化を実現する。
```

## 1.2　システム構成
```
ユーザー（ブラウザ）
  ↓ HTTP (localhost:8501)
Streamlit（GUI）
  ↓ HTTP (localhost:8000)
FastAPI（APIサーバー）
  ↓ HTTPS（Gemini API通信）
Gemini API

FastAPI
  ↓
ローカル JSON ファイル（データ保存）
```

| レイヤー | 技術 | 用途 |
|---|---|---|
| フロントエンド | Streamlit | 画面表示・ユーザー操作 |
| バックエンド | FastAPI | APIエンドポイント・業務ロジック |
| AI | Gemini API (google-genai) | カテゴリ分類・緊急度判定・回答案生成 |
| データ保存 | JSON ファイル | タスクデータの永続化 |
| 仮想環境 | uv | Pythonパッケージ管理 |
| バージョン管理 | Git / Sourcetree | ソースコード管理 |

## 2. 機能要件
### 2.1 問い合せ入力機能
- ユーザーがテキストエリアに問い合わせ内容を入力できる
- 問い合わせ内容が未入力の場合はエラーメッセージを表示する
- 送信後に送信完了メッセージを表示すること

### 2.2 AIカテゴリー判定機能
- 問い合わせ内容を Gemini AI でカテゴリー判定して値を返す
- AI がカテゴリーを以下から1つ選択する
  - 勤怠
  - 休暇
  - 給与
  - 経費精算
  - 社員情報変更
  - その他

### 2.3 AI緊急度判定機能
- AIが問い合わせ内容から緊急度を判定する
- 緊急度は以下の3段階とする
  - 高
  - 中
  - 低

### 2.4 AI回答案生成機能
- AIが問い合わせ内容に対する回答案を生成する
- 生成した回答案を画面へ表示する

### 2.5 履歴保存機能
- 問い合わせ内容を JSON ファイルへ保存する
- 保存項目は以下とすること
  - 問い合わせ内容
  - カテゴリ
  - 緊急度
  - 回答案
  - 登録日時

### 2.6 一覧表示機能
- 保存済み問い合わせを一覧表示できる
- 一覧には以下を表示する
  - 問い合わせ内容
  - カテゴリ
  - 緊急度
  - 回答案
  - 登録日時

### 2.7 詳細表示機能
- 問い合わせ詳細を表示できる
- 詳細画面には以下を表示する
  - 問い合わせ内容
  - カテゴリ
  - 緊急度
  - 回答案
  - 登録日時

## 5. API 一覧（FastAPI）

| メソッド | パス | 機能 |
|---|---|---|
| POST | `/inquiries` | 問い合わせを登録する |
| GET | `/inquiries` | 問い合わせ一覧を取得する |
| GET | `/inquiries/{id}` | 問い合わせ詳細を取得する |

## 6 データファイル仕様

- ファイル名：`inquiries.json`
- 保存場所：`data/inquiries.json`
- フォーマット：JSON配列
- 文字コード：UTF-8

```json
[
  {
    "id": 1,
    "created_at": "2026-05-14T10:00:00+09:00",
    "question": "有給休暇の申請方法を教えてください。",
    "category": "休暇",
    "urgency": "低",
    "answer": "有給休暇の申請は..."
  }
]

```

| フィールド名 | 型 | 内容 |
|---|---|---|
| id | 整数 | 問い合わせID（自動採番） |
| created_at | 文字列（ISO 8601） | 登録日時 |
| question | 文字列 | 問い合わせ本文 |
| category | 文字列 | AIが判定したカテゴリ（ 勤怠／休暇／給与／経費精算／社員情報変更／その他） | 
| urgency | 文字列 | AIが判定した緊急度（高／中／低） |
| answer | 文字列 | AIが生成した回答案 |

## 7. 対象
## 7.1 対象内
### 新規問い合わせ入力できる
- 社員はテキストエリアから新規問い合わせ内容を入力できる。
  入力項目には、問い合わせ内容を含む。

### エラー表示機能
- 問い合わせ入力時に必須項目が未入力の場合、エラーメッセージを表示する。
  利用者が入力不足を認識できるよう、画面上で未入力を通知する。

### 問い合わせ一覧表示機能
- 入力済みの問い合わせを一覧で表示できる。
  一覧画面では、問い合わせのカテゴリ・緊急度・回答案などを一通り確認可能とする。

### 詳細表示機能
- 問い合わせ一覧から任意の問い合わせの詳細情報を表示できる。
  詳細画面では、問い合わせ内容・AI 生成結果・登録日時などをすべて表示可能とする。

### AI 生成機能
- 問い合わせ入力時に AI が問い合わせ内容を解析し、以下の情報を自動生成する。
  問い合わせカテゴリ
  緊急度
  回答案
  AI 生成には Gemini API を利用する。

### API キー管理機能
- Gemini API で使用する API キーは、ソースコードへ直接記述しない。
  API キーは環境変数等を用いて安全に管理する。

### データ永続化機能
- 問い合わせ内容および AI 生成結果を JSON ファイルへ書き込み、アプリケーション終了後もデータを保持できる。
- 永続的にデータを読み書きできる

### 履歴参照機能
- 過去に入力された問い合わせ履歴を参照できる。
  これにより、同様の問い合わせに対する過去対応内容を再利用可能とする。

### ローカル実行対応
- 本システムはローカル PC 環境上で動作する。
  Web ブラウザを利用してシステムへアクセス可能とする。

## 7.2 対象外
### 総務担当者向け管理機能
- 問い合わせの担当者割当、ステータス管理、対応進捗管理などの管理者機能は対象外とする。

### 認証・ログイン機能
- ユーザー認証、ログイン、アカウント管理機能は実装対象外とする。

### 排他制御
- 複数ユーザーによる同時編集時の排他制御および整合性管理は対象外とする。

### クラウドデプロイ
- AWS、Azure 等のクラウド環境へのデプロイは対象外とする。

### メール通知機能
- 回答時のメール通知機能は実装対象外とする。

### データベース利用
- MySQL および NoSQL の導入は行わず、データ保存は JSON ファイルで管理する。
