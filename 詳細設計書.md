## 1. API設計（各エンドポイントのリクエスト・レスポンスの詳細）
## 1.1. 各メソッドの機能

| メソッド | パス | 機能 |
|---|---|---|
| POST | `/inquiries` | 問い合わせ登録・保存 |
| GET | `/inquiries` | 問い合わせ一覧を取得する |
| GET | `/inquiries/{id}` | 問い合わせ詳細取得 |

### 1.2. POST /inquiries 問い合わせ登録
### 概要： タスク情報を受け取り、JSONファイルに保存する。
```
HTTPメソッド  --> POST
URL          --> http://localhost:8000/inquiries
Content-Type --> application/json
```

```json
[
  {
    "question": "有給休暇の申請方法を教えてください。",
    "category": "休暇",
    "urgency": "低",
    "answer": "有給休暇の申請は..."
  }
]
```

### 各フィールドの型・説明

| フィールド名 | 型           | 説明          | 必須 |
|---|---|---|---|
| content     | string       | 問い合わせ内容 （空不可） |  Yes  |
| category    | string       | 勤怠 / 休暇 / 給与 / 経費精算 / 社員情報変更  |  Yes  |
| urgency     | string       | 高 / 中 / 低 |  Yes  |
| answer      | string       | 回答案 |  Yes  |

### 成功レスポンス（201）

```json
[
  {
    "id": 1,    #連番整数ID
    "created_at": "2026-05-15T10:00:00+09:00",
    "question": "有給休暇の申請方法を教えてください。",
    "category": "休暇",
    "urgency": "低",
    "answer": "有給休暇の申請は..."
  }
]
```

### 1.3. GET /inquiries 問い合わせ一覧取得
### 概要： 保存済みの問い合わせ内容を全件、登録時刻の降順で返す。
```
HTTPメソッド  --> GET
URL          --> http://localhost:8000/inquiries
```

### 仕様

- 登録時刻（created_at）の降順で返す
- 件数が0件のときは空のリスト [ ] を返す

## 2. データモデル（Pydantic）
ファイル：`backend/main.py`
```python
from pydantic import BaseModel

class AnalysisResult(BaseModel):
    question: str
    category: str
    urgency: str

class InquiryRequest(BaseModel):
    id: int
    created_at: str
    question: str
    category: str
    urgency: str
    answer: str

```
| クラス | 用途 |
|---|---|
| TaskRequest | フロントエンドから受け取るPOSTリクエストのボディ |
| TaskRecord  | JSON ファイルに保存・APIから返却するデータ |
