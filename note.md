### 新規問い合わせ入力できる
- 社員はテキストエリアから新規問い合わせ内容を入力できる。
  入力項目には、問い合わせ内容を含む。

### エラー表示機能
- 問い合わせ入力時に必須項目が未入力の場合、エラーメッセージを表示する。
  利用者が入力不足を認識できるよう、画面上で未入力を通知する。

### 問い合わせ一覧表示機能
- 入力済みの問い合わせを一覧で表示できる。
  一覧画面では、問い合わせのカテゴリ・緊急度・回答案などを一通り確認可能とする。

### 詳細表示機能
- 問い合わせ一覧から任意の問い合わせの詳細情報を表示できる。
  詳細画面では、問い合わせ内容・AI 生成結果・登録日時などをすべて表示可能とする。

### AI 生成機能
- 問い合わせ入力時に AI が問い合わせ内容を解析し、以下の情報を自動生成する。
  問い合わせカテゴリ
  緊急度
  回答案
  AI 生成には Gemini API を利用する。

### API キー管理機能
- Gemini API で使用する API キーは、ソースコードへ直接記述しない。
  API キーは環境変数等を用いて安全に管理する。

### データ永続化機能
- 問い合わせ内容および AI 生成結果を JSON ファイルへ書き込み、アプリケーション終了後もデータを保持できる。
- 永続的にデータを読み書きできる

### 履歴参照機能
- 過去に入力された問い合わせ履歴を参照できる。
  これにより、同様の問い合わせに対する過去対応内容を再利用可能とする。

### ローカル実行対応
- 本システムはローカル PC 環境上で動作する。
  Web ブラウザを利用してシステムへアクセス可能とする。

## 7.2 対象外
### 総務担当者向け管理機能
- 問い合わせの担当者割当、ステータス管理、対応進捗管理などの管理者機能は対象外とする。

### 認証・ログイン機能
- ユーザー認証、ログイン、アカウント管理機能は実装対象外とする。

### 排他制御
- 複数ユーザーによる同時編集時の排他制御および整合性管理は対象外とする。

### クラウドデプロイ
- AWS、Azure 等のクラウド環境へのデプロイは対象外とする。

### メール通知機能
- 回答時のメール通知機能は実装対象外とする。

### データベース利用
- MySQL および NoSQL の導入は行わず、データ保存は JSON ファイルで管理する。
